#Node.js:
-It is a JavaScript run time environment that allows us to run JS code outside of a web browser.
-It is an open source cross platform JavaScript run time environment that runs on chrome's v8 engine.
-Node.js is known for its non-blocking I/o and event- driven architecture making it great for building scalable and high performance application.

#Node.js Architecture:
1.Event-Driven/Single-threaded:
-It handles multiple connection using a single thread via the event loop and callback.

2.Non blocking I/O:
-Operation like reading files databases or network calls do not block execution.
-Instead callback, promises, asyn-await handles complition.

#Node REPL(read-evaluate-print-loop):
-It is an interative shell in Node.js that allows to execute JavaScript code line by line and see immediate result.
-It is useful for quick testing, debbuging and learning where JavaScript code can be experimented with directly.

#How it works?
1.Read- It takes the user input.
2.Evaluate- It evaluate the code immediately in the same line.
3.Print- It shows the output or result.
4.Loop- It waits for the next input until the user exit.

-To start Node repl open terminal or command promt then type node press enter.
-To exit we can use .exit and to clear the environment .clear 

#What are module in Node.js?
-A module in Node.js is a self-contained unit of code placed in a file or folder, capable of exporting functionalities for use elsewhere.
-Modules prevent global namespace pollution help organising code and improve maintainability.

-There are 3 types of Modules:
1.Built-in Modules- Included with Node.js
2.Local modules- User defined or created by developers as a part of project
3.External modules- Install via npm and stored in 'node_modules' folder

#Module System in Node.js-
-Node.js uses powerful module system to help organise code, promote, re-use and encapsulate functionalities of Node.js application structure.
-There are two primary systems in node.js
1.Common js Modules
2.ES6/ECMA script modules

1)Common js modules system:
-It is a traditional module system in node.js
-It uses require() method to import modules and "module.exports" or exports to export functionalities.
-Each module or file is wrap in a function exposing module, export, required __filename, __directoryname for local scope ensuring variables do not leak into global scope.

2)ES6/ECMA Script module system:
-It is a modern standardised module utilising "import and export ".
-Modules are loaded asynchronously for better performance.
-We have to use "type":"module" in package.json to enable ES6. (if there is no package.json then use .mjs extension)

Node.js FileSystem(fs):
--Node.js filesystem(fs) module enables file and directory I/O operations offering both sysnchronous and asynchronous methods to handle files efficiently in different scenario.

Use of fs module :
--The Node.js fs module is a core part of Node.js, that allows programm to interact with the computer's filesystem.
--Its primary use is to perform essential file and diectory Operations such as reading writing creating deleting and monitoring files. 

Synchronous Method :
--Sysnchronous method block execution until the operation completes, making other process wait, which can be inefficient for I/O bound task.
--Some common sysnchronous method :-
     ---fs.readFileSync()
     ---fs.appendFileSync()
     ---fs.mkdirSync()
     ---fs.existsSync()
     ---fs.writeFileSync()
     ---fs.renameSync()

Asynchronous Method :
--Asynchronous method do not block execution instead they use callbacks, promises or async-await allowing other Operations to run curently and improving for heavy I/O workload.
--These are preffered for scalable high-performance application or server
--Some common sysnchronous method :-
     ---fs.unlink()
     ---fs.readFile()
     ---fs.appendFile()
     ---fs.exits()
     ---fs.rename()


HTTP Server(Hyper Text Transfer Protocol):
--An HTTP server is programm that listens for request from clients and sends back response(like web page or data).
--In Node.js we can easily create an HTTP server using the builtin HTTP module.
--client or browser sends a request like URL,then server listens on a specific port no and then the server processes the request and sends back a responds like HTML, text or json

http module ---> http server --->request --->response
         
          = request----->http server--->response

Steps to create https server in Node.js :-
Step 1: we have to import http module  either by using require() or import statement.

const http=require('http')
--The http module provides methods to create and manage servers.

Step 2: create a server by using creatServer()

const server = http.creatServer((req,res)=>{
     res.writeHead(200,('Content-Type': 'text/plain'));
     res.end("message");
})

req-> Request object which contains data from client side
res-> Response object which is used to send data to client
writeHead()->it is a method which is used to define the header or set the status code and content type.
end()->it is is a method which is used to send data to client side

Step 3: Listen/Start the server by using listen()
--we need to pass a port  number as an argument to the listen() to run the server on a specific port.

server.listen(3000,()=>{
     log("Server running at http://localhost:3000")
})

--Port number can be anything in between 1024 and 65555(e.g: 3000,5000,8000)
--Now we can open our browser and go to the URL to see the Response


Some Important Request Properties
req.url : the path requested by client side
e.g : /about , /contact

req.method : it is the http method used to check request type 
e.g: GET, POST, PUT, DELETE

req.headers : it contains all request headers
e.g : {host : 'localhost.....'} 

Important Response Properties :
res.writeHead(statusCode,header): It is used to set the HTTP status code and response header.

res.write(data) : It is used to send data in Chunks

res.end([data]) : sends response, and it must be called


COMMON HTTP STATUS CODE
200 -> ok -> Request successful
201 -> created -> Data created
400 -> Bad Request -> Invalid Request 
401 -> Unauthorized -> No Authentication
403 -> Forbidden -> Access denied
404 -> Not found -> Page Not found
500 -> Internal server error -> server crashed
